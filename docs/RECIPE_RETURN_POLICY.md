# Recipe Return Policy

## السياسة المعتمدة

مرتجع المنتج الذي له Recipe لا يعيد المكونات إلى المخزون، ولا يعكس COGS.

## السبب

عند بيع منتج له Recipe يتم استهلاك المكونات الفعلية مثل العجين أو الجبن. عند رجوع المنتج النهائي من الزبون لا يمكن افتراض أن المكونات عادت بحالة قابلة للبيع أو الاستخدام. لذلك لا يتم تسجيل زيادة في `inventory_stock` ولا يتم إنشاء حركة `RETURN` للمكونات.

## الأثر المحاسبي

مرتجع Recipe يعكس الإيراد والنقد فقط:

```txt
Dr Sales Revenue
Cr Cash
```

ولا يتم إنشاء سطور:

```txt
Dr Inventory
Cr COGS
```

## الأثر المخزني

- لا يتم إرجاع المكونات حسب BOM.
- لا يتم إرجاع المنتج النهائي إلى المخزون.
- تبقى حركات `RECIPE_CONSUMPTION` كما هي.

## قاعدة الاتساق

المحاسبة لا تسجل زيادة مخزون إذا كان `inventory_stock` لم يزد. هذا يمنع فرقاً بين دفتر المخزون والقيود المالية.
