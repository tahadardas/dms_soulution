# مسار عمل POS

## فتح الوردية

1. يسجل الكاشير الدخول.
2. يفتح وردية.
3. يدخل مبلغ الافتتاح.
4. يبدأ البيع.

Endpoint:

```http
POST /pos/sessions/open
```

## بيع عادي

1. اختر نوع الطلب: صالة، سفري، دليفري.
2. اختر المنتجات.
3. اختر طريقة الدفع عند الدفع الفوري.
4. احفظ الطلب.
5. اطبع عند الحاجة من زر الطباعة.

حفظ الطلب لا ينشئ طباعة تلقائية.

Endpoint:

```http
POST /pos/orders
```

## الدفع

- CASH يدخل الصندوق.
- CARD يرحل إلى البنك ولا يدخل الصندوق.
- TRANSFER يرحل إلى البنك ولا يدخل الصندوق.

## المرتجع

1. اختر الطلب.
2. اختر البنود والكميات.
3. أدخل السبب.
4. نفذ المرتجع.

Endpoint:

```http
POST /pos/returns
```

## الإلغاء

الإلغاء يحتاج سبباً. الطلب المدفوع أو المطبوع قد يحتاج موافقة مدير حسب السياسة.

Endpoint:

```http
POST /pos/orders/:orderId/void
```

## إغلاق الوردية

1. أدخل النقد الفعلي.
2. إذا كان هناك فرق، أدخل السبب.
3. إذا كان الفرق كبيراً، أدخل بيانات المدير.
4. أغلق الجلسة.

Endpoint:

```http
POST /pos/sessions/close
```
