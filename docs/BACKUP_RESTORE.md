# النسخ الاحتياطي والاستعادة

النسخ الاحتياطي جزء من التشغيل اليومي، وليس إجراء طارئاً فقط.

## المسارات

المسار الافتراضي:

```txt
<working-directory>\backups
```

يمكن تغييره من صفحة إدارة النسخ الاحتياطية أو عبر إعداد النظام.

## إنشاء backup يدوي

من الواجهة:

```txt
Settings -> Backups -> Take Backup Now
```

من API:

```http
POST /admin/backups
```

يتطلب صلاحية إدارة الإعدادات.

## الاستعادة

من API:

```http
POST /admin/backups/restore
```

قواعد الاستعادة:

- لا تتم الاستعادة إذا كانت هناك جلسة POS مفتوحة.
- لا تتم الاستعادة من ملف خارج مجلد النسخ الاحتياطي.
- بعد الاستعادة يجب إعادة تشغيل التطبيق.
- يجب تجربة الاستعادة على نسخة تدريبية قبل استخدامها في الإنتاج.

## الحذف

```http
DELETE /admin/backups/:filename
```

الحذف يقبل اسم ملف فقط، وليس مساراً كاملاً.

## سياسة مقترحة

- backup يدوي قبل بداية اليوم.
- backup يدوي بعد نهاية اليوم.
- حفظ آخر 14 نسخة يومية.
- نسخ مجلد backups إلى وسيط خارجي مرة يومياً.

## فحص سلامة النسخة

بعد إنشاء backup مهم:

1. انسخ الملف إلى جهاز اختبار.
2. شغل API باستخدام `DMS_DB_PATH` يشير إلى النسخة.
3. نفذ:

```bash
npm run db:verify --workspace=@dms/api
```
