# سجل المخاطر

## R1 - ثغرات npm

الحالة: مغلق حالياً.

الوصف: `npm install` أظهر سابقاً ثغرات، وتمت ترقية Fastify وVite وElectron وبعض التبعيات العابرة.

التحقق: `npm audit --audit-level=moderate` يعيد 0 vulnerabilities بتاريخ 2026-05-09.

الإجراء الدوري: إعادة تشغيل audit قبل كل إصدار.

## R2 - اختبار الطابعات الفعلية

الحالة: مفتوح.

الوصف: البناء والاختبارات البرمجية لا تثبت أن طابعات المطعم الفعلية تعمل.

الإجراء: اختبار إيصال وKOT على شبكة المطعم.

## R3 - Restore في الإنتاج

الحالة: مخفف.

الإصلاح: منع الاستعادة أثناء وجود جلسات مفتوحة ومنع أسماء ملفات خارج مجلد النسخ.

الإجراء المتبقي: تجربة الاستعادة على جهاز اختبار قبل اعتمادها.

## R4 - RTL والواجهة العربية

الحالة: مفتوح جزئياً.

الوصف: النصوص العربية موجودة، لكن يلزم اختبار بصري كامل على شاشات الكاشير.

الإجراء: تنفيذ `FULL_PRODUCTION_MANUAL_TEST.md` بالعربية.

## R5 - تشغيل API كخدمة

الحالة: مفتوح.

الوصف: الكود جاهز للبناء، لكن جهاز المطعم يحتاج إعداد Windows Service أو PM2.

الإجراء: تطبيق `DEPLOYMENT_RESTAURANT_PC.md`.

## R6 - انقطاع الكهرباء

الحالة: مفتوح.

الإجراء: استخدام UPS، تفعيل backups يومية، وعدم وضع قاعدة البيانات على قرص شبكة غير مستقر.

## R7 - صلاحيات المدير

الحالة: مخفف.

الإصلاح: فروقات الصندوق والإلغاء الحساس تدعم موافقة مدير.

الإجراء المتبقي: عدم مشاركة حساب المدير مع الكاشيرين.
